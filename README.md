# Healthcare Appointment Management System

## Overview
A relational database solution designed to optimize the scheduling and management of patient appointments within a medical facility. Developed during my internship at a health foundation, this project showcases my ability to design schemas for the Digital Health sector.

## Key Features
- **Patient Registration:** Structured storage of demographics and contact information.
- **Doctor Scheduling:** Management of medical staff, specialties, and availability.
- **Appointment Booking:** Logic to link patients with specific doctors, dates, and visit reasons.
- **Status Tracking:** Monitoring appointment states (e.g., Scheduled, Completed, Cancelled).

## Technical Skills Demonstrated
- **Relational Data Modeling:** Using Primary and Foreign Keys to link patients, doctors, and appointments.
- **Data Integrity Constraints:** Ensuring data accuracy (e.g., unique patient IDs).
- **Business Logic via Queries:** Developing SQL queries for operational reporting, such as listing a doctor's schedule or identifying cancelled appointments.

## Case Study Focus
This project simulates real-world healthcare administration needs, emphasizing efficiency and organized data access in a medical environment.
